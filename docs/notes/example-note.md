# Диск переполнен, а du не находит крупные файлы

## Проблема

`df -h` показывает 100% занятого места на разделе, но `du -sh /*` не находит файлов, которые объясняют такой объём.

## Причина

Файл был удалён, но его всё ещё держит открытым какой-то процесс (файловый дескриптор). Место освобождается только после того, как процесс закроет дескриптор или завершится.

## Решение

Найти процессы, держащие удалённые файлы:

```bash
lsof +L1 | grep deleted
```

Или через `/proc`:

```bash
find /proc/*/fd -ls 2>/dev/null | grep '(deleted)'
```

Если файл действительно не нужен — перезапустить или завершить процесс, который его держит (например, `systemctl restart <service>`), либо очистить его вывод через дескриптор:

```bash
> /proc/<PID>/fd/<FD>
```

## Как избежать в будущем

- Настроить `logrotate` для сервисов с большими логами.
- Мониторить `df -h` через Zabbix/Prometheus с алертом на 80%+.

!!! tip "Быстрая проверка"
    `du -sh /* 2>/dev/null | sort -rh | head` — если сумма сильно меньше показаний `df`, ищи удалённые, но открытые файлы.
