# Инструкция: включение отладки и смена пути каталога `srvinfo` сервера 1С:Предприятие

## Назначение

Данная инструкция описывает:

- включение серверной отладки 1С;
- перенос каталога `srvinfo` (служебные данные сервера, журналы регистрации, кэш и настройки кластера);
- проверку корректности работы после внесения изменений.

!!! warning "Важно"

    Перед изменением параметров сервера рекомендуется сделать резервную копию каталога `srvinfo`.

---

## 1. Включение серверной отладки

### Шаг 1. Остановить службу сервера 1С

Откройте **Службы** (`services.msc`) и остановите службу:

```
1C:Enterprise 8.3 Server Agent
```

или выполните:

```cmd
net stop "1C:Enterprise 8.3 Server Agent"
```

### Шаг 2. Добавить параметр `-debug`

Откройте редактор реестра:

```
regedit
```

Перейдите в раздел:

```
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\1C:Enterprise 8.3 Server Agent
```

Откройте параметр:

```
ImagePath
```

Добавьте в конец строки параметр:

```text
-debug
```

#### Пример

До изменения:

```text
"C:\Program Files\1cv8\8.3.xx.xxxx\bin\ragent.exe" -srvc -agent -regport 1541 -port 1540 -range 1560:1591 -d "C:\Program Files\1cv8\srvinfo"
```

После изменения:

```text
"C:\Program Files\1cv8\8.3.xx.xxxx\bin\ragent.exe" -srvc -agent -regport 1541 -port 1540 -range 1560:1591 -d "C:\Program Files\1cv8\srvinfo" -debug
```

### Шаг 3. Запустить службу

```cmd
net start "1C:Enterprise 8.3 Server Agent"
```

или через **services.msc**.

### Шаг 4. Проверка

Откройте **Конфигуратор**.

Перейдите:

```
Отладка → Подключение
```

Если всё выполнено правильно:

- будут отображаться серверные сеансы;
- точки останова серверного кода начнут срабатывать.

---

## 2. Перенос каталога `srvinfo`

### Что такое `srvinfo`

Каталог `srvinfo` хранит:

- журналы регистрации;
- описание кластера;
- настройки информационных баз;
- кэш сервера;
- служебные файлы.

По умолчанию каталог находится здесь:

```
C:\Program Files\1cv8\srvinfo
```

### Шаг 1. Остановить службу

```cmd
net stop "1C:Enterprise 8.3 Server Agent"
```

### Шаг 2. Создать новый каталог

Например:

```
D:\1C\srvinfo
```

или

```
E:\1C\srvinfo
```

### Шаг 3. Скопировать содержимое

Полностью скопируйте содержимое старого каталога:

```
C:\Program Files\1cv8\srvinfo
```

в новый каталог:

```
D:\1C\srvinfo
```

Лучше использовать:

```cmd
robocopy "C:\Program Files\1cv8\srvinfo" "D:\1C\srvinfo" /MIR /COPYALL
```

### Шаг 4. Изменить параметр `-d`

Откройте реестр:

```
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\1C:Enterprise 8.3 Server Agent
```

Измените значение параметра **ImagePath**.

#### Было

```text
-d "C:\Program Files\1cv8\srvinfo"
```

#### Стало

```text
-d "D:\1C\srvinfo"
```

Полностью строка может выглядеть так:

```text
"C:\Program Files\1cv8\8.3.xx.xxxx\bin\ragent.exe" -srvc -agent -regport 1541 -port 1540 -range 1560:1591 -d "D:\1C\srvinfo"
```

Если включена отладка:

```text
"C:\Program Files\1cv8\8.3.xx.xxxx\bin\ragent.exe" -srvc -agent -regport 1541 -port 1540 -range 1560:1591 -d "D:\1C\srvinfo" -debug
```

### Шаг 5. Запустить службу

```cmd
net start "1C:Enterprise 8.3 Server Agent"
```

### Шаг 6. Проверка

Проверьте:

- служба успешно запустилась;
- кластер отображается в консоли администрирования;
- информационные базы доступны;
- пользователи успешно подключаются;
- в новом каталоге появляются новые служебные файлы.

---

## Возможные проблемы

### Служба не запускается

Проверьте:

- правильность пути после параметра `-d`;
- наличие каталога;
- права доступа службы к новому каталогу.

### Кластер не отображается

Чаще всего причиной является:

- пустой каталог вместо скопированного;
- повреждение содержимого `srvinfo`;
- неверный путь в `ImagePath`.

### Не срабатывает отладка

Проверьте:

- присутствует ли параметр `-debug`;
- перезапущена ли служба;
- подключена ли конфигурация в режиме отладки.

---

## Рекомендации

- Не храните каталог `srvinfo` на системном диске при высокой нагрузке.
- Используйте SSD или NVMe.
- Перед переносом обязательно создавайте резервную копию.
- Не удаляйте старый каталог до успешной проверки работы сервера.
- Включайте параметр `-debug` только на время разработки или диагностики, так как он может снизить производительность.

---

## Быстрая памятка

| Задача | Что изменить |
|---------|--------------|
| Включить серверную отладку | Добавить параметр `-debug` |
| Перенести каталог `srvinfo` | Изменить путь после параметра `-d` |
| Применить изменения | Перезапустить службу `1C:Enterprise 8.3 Server Agent` |
| Проверить работу | Открыть Конфигуратор или консоль кластера 1С |
