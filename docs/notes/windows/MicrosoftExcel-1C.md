# Настройка DCOM для Microsoft Excel Application (1С)

Данная инструкция описывает настройку разрешений DCOM для компонента **Microsoft Excel Application**, используемого при автоматизации Microsoft Excel из **1С**.

> **Важно:** Все действия необходимо выполнять от имени администратора.

---

## 1. Открытие служб компонентов

Откройте окно **Выполнить** (`Win + R`) и выполните команду:

```cmd
dcomcnfg
```

Если компонент **Microsoft Excel Application** отсутствует в списке, откройте 32-битную оснастку:

```cmd
mmc comexp.msc /32
```

Перейдите по следующему пути:

```text
Component Services
└── Computers
    └── My Computer
        └── DCOM Config
```

Найдите компонент:

```text
Microsoft Excel Application
```

---

## 2. Если Microsoft Excel Application отсутствует

Проверьте наличие раздела реестра:

```text
HKEY_CLASSES_ROOT\AppID\EXCEL.EXE
```

Если раздел отсутствует:

1. Создайте раздел:

   ```text
   EXCEL.EXE
   ```

2. Установите значение **(По умолчанию)** типа **REG_SZ**:

   ```text
   {00020812-0000-0000-C000-000000000046}
   ```

3. Повторно выполните команду:

   ```cmd
   mmc comexp.msc /32
   ```

После этого компонент **Microsoft Excel Application** должен появиться в списке DCOM.

> Также можно импортировать готовый файл реестра [Скачать **MicrosoftExcelApplication.reg**](../files/MicrosoftExcelApplication.reg), после чего повторно открыть оснастку DCOM.

---

## 3. Настройка разрешений Launch and Activation

Откройте **Свойства** компонента **Microsoft Excel Application** и перейдите на вкладку **Security**.

В разделе **Launch and Activation Permissions**:

1. Выберите **Customize**.
2. Нажмите **Edit**.
3. Добавьте учетную запись, от имени которой запускается служба **1С**.
4. Предоставьте следующие разрешения:
   - ✅ Local Launch
   - ✅ Local Activation

---

## 4. Настройка Access Permissions

В том же окне перейдите к разделу **Access Permissions**.

1. Выберите **Customize**.
2. Нажмите **Edit**.
3. Добавьте ту же учетную запись службы **1С**.
4. Предоставьте разрешение:
   - ✅ Local Access

Нажмите **OK** для сохранения изменений.

---

## 5. Проверка каталогов Desktop

Убедитесь, что существуют следующие каталоги:

```text
C:\Windows\System32\config\systemprofile\Desktop
```

```text
C:\Windows\SysWOW64\config\systemprofile\Desktop
```

Если какого-либо каталога нет — создайте его.

---

## 6. Настройка прав на каталоги

Для обеих папок предоставьте учетной записи, под которой работает служба **1С**, следующие права:

- ✅ Полный доступ (Full Control)

---

## Проверка работы

После завершения настройки рекомендуется:

1. Перезапустить службу **1С**.
2. При необходимости перезагрузить сервер.
3. Проверить выполнение операций, использующих Microsoft Excel:
   - экспорт в Excel;
   - формирование печатных форм;
   - обработку Excel-файлов.

---

## Возможные проблемы

### Microsoft Excel Application отсутствует

Убедитесь, что:

- выполнена команда:

  ```cmd
  mmc comexp.msc /32
  ```

- существует раздел реестра:

  ```text
  HKEY_CLASSES_ROOT\AppID\EXCEL.EXE
  ```

---

### Ошибка доступа при работе с Excel

Проверьте:

- настройки разрешений DCOM;
- права на каталоги `Desktop`;
- учетную запись, от имени которой запущена служба **1С**.

---

## Итог

После выполнения описанных действий:

- Microsoft Excel сможет запускаться из службы **1С**;
- исчезнут ошибки DCOM при автоматизации Excel;
- экспорт, импорт и обработка Excel-файлов из **1С** будут выполняться корректно.
